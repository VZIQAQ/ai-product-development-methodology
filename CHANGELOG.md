# Changelog / 变更日志

All notable changes to this methodology will be documented in this file.

## [1.0.0] - 2026-07-14

### Added / 新增
- Initial release of Spec-Driven AI Product Development Methodology
- Core principles: natural language specification, Agent-assisted derivation
- Strict serial iteration: lock Spec → develop → deliver → next round
- Test-first approach: Agent generates tests, user confirms coverage
- Delivery handover document for continuous improvement
- Bilingual documentation (English/Chinese)

- Spec驱动AI产品开发方法论初始发布
- 核心原则：自然语言规范、Agent辅助推导
- 严格串行迭代：锁定Spec → 开发 → 交付 → 下一轮
- 测试前置：Agent生成测试、用户确认覆盖度
- 交付交接文档用于持续改进
- 双语文档（英文/中文）

### Methodology Highlights / 方法论亮点
- No tool recommendations - methodology over tools / 无工具推荐 - 方法论高于工具
- Human responsible for business logic, AI handles technical details / 人负责业务逻辑，AI处理技术细节
- Clear boundaries: execution deviation vs Spec omission vs new requirements / 清晰边界：执行偏差 vs Spec遗漏 vs 新需求
- Context isolation for parallel development / 上下文隔离用于并行开发
