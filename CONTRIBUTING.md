# Contributing to AI Product Development Methodology / 贡献指南

Thank you for your interest in contributing! This project welcomes ideas, corrections, and translations.

感谢你的贡献兴趣！本项目欢迎想法、修正和翻译。

## How to Contribute / 如何贡献

### Reporting Issues / 报告问题
- Use GitHub Issues to report problems or suggest improvements
- Clearly describe the issue with examples
- Include your context (personal project, team size, AI tools used)

- 使用 GitHub Issues 报告问题或建议改进
- 用示例清楚描述问题
- 包含你的上下文（个人项目、团队规模、使用的AI工具）

### Proposing Changes / 提议变更
1. Fork the repository / Fork 仓库
2. Create a feature branch (`git checkout -b feature/your-idea`) / 创建功能分支
3. Make your changes / 做出你的修改
4. Submit a Pull Request with clear description / 提交带有清晰描述的 PR

### Content Guidelines / 内容指南
- Keep methodology language-agnostic (no tool-specific recommendations)
- Provide concrete examples for abstract concepts
- Maintain the core principle: human judges, AI executes

- 保持方法论与语言无关（不推荐具体工具）
- 为抽象概念提供具体示例
- 保持核心原则：人判断，AI执行

### Translation / 翻译
- We welcome translations to other languages
- Please maintain the bilingual structure (methodology terms in both languages)
- Open an issue first to coordinate translation efforts

- 欢迎翻译成其他语言
- 请保持双语结构（方法论术语用两种语言）
- 先开 Issue 协调翻译工作

## Code of Conduct / 行为准则

- Be respectful and constructive / 保持尊重和建设性
- Focus on methodology improvement, not tool debates / 聚焦方法论改进，不是工具争论
- Acknowledge that different contexts require different approaches / 承认不同场景需要不同方法
